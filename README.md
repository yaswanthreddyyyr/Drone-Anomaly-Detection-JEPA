Score = 1.60 × AUC + 2.30 × Recall + 0.20 × F1 − 3.20 × max(0, FAR − 0.20)
```
Recall is weighted highest because missing a real attack is worse than a false alarm.

## 🧠 How It Works

Drone telemetry tampering (GPS spoofing, altitude spikes, coordinate jumps, etc.) can cause route deviation and mission failure. Manual monitoring doesn't scale. This project uses a self-supervised approach — train only on normal flights, then flag deviations at inference time.

**Three-stage pipeline:**

1. **JEPA Encoder** — Takes 11-feature telemetry windows (chunk size 20, stride 10), randomly masks 12–72% of the input, and trains to predict the masked content in embedding space — not in raw feature space. This forces the model to learn the underlying structure of normal drone behavior. Trained on normal flights only.

2. **LOF Detector** — Local Outlier Factor scores each 384-dim JEPA embedding by comparing its local density to k nearest neighbors. Normal samples cluster densely; anomalies sit in sparse, isolated regions.

3. **Threshold Calibration** — The decision threshold is tuned on the validation set to maximize the composite score under a strict 20% FAR cap.

**Champion detector config:** `StandardScaler → PCA(130 components) → LOF(Manhattan distance, k=21)`

import pickle, torch, json
import numpy as np
from pathlib import Path
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.metrics import classification_report, confusion_matrix, roc_auc_score
from torch.utils.data import DataLoader
from src.data.dataset import DroneChunkDataset
from src.models.jepa import JEPA
from src.models.isolation_forest import EmbeddingExtractor
from google.colab import drive
import shutil, yaml

drive.mount('/content/drive')

# Copy models from Drive into the project outputs folder
shutil.copy("/content/drive/MyDrive/best_model.pt",
            "/content/Drone-Anomaly-Detection-JEPA/outputs/best_model.pt")
shutil.copy("/content/drive/MyDrive/best_lof_model.pkl",
            "/content/Drone-Anomaly-Detection-JEPA/outputs/best_lof_model.pkl")
shutil.copy("/content/drive/MyDrive/lof_local_refinement_summary.json",
            "/content/Drone-Anomaly-Detection-JEPA/outputs/lof_local_refinement_summary.json")

# Load config and reconstruct the model
with open("configs/config.aggressive_v3.yaml") as f:
    cfg = yaml.safe_load(f)

device = "cuda" if torch.cuda.is_available() else "cpu"
train_ds = DroneChunkDataset("processed_data", split="train", return_labels=True)
train_normal = train_ds.get_normal_chunks()

model = JEPA(
    input_dim=train_ds.num_features,
    embed_dim=cfg["model"]["embedding_dim"],
    encoder_hidden=cfg["model"]["encoder_hidden"],
    predictor_hidden=cfg["model"]["predictor_hidden"],
    dropout=cfg["model"]["dropout"],
    min_mask_ratio=cfg["masking"]["min_mask_ratio"],
    max_mask_ratio=cfg["masking"]["max_mask_ratio"],
)
state = torch.load("outputs/best_model.pt", map_location="cpu")
model.load_state_dict(state["model_state_dict"] if "model_state_dict" in state else state)
model.eval()

# Load LOF detector and calibrated threshold
with open("outputs/best_lof_model.pkl", "rb") as f:
    lof = pickle.load(f)
with open("outputs/lof_local_refinement_summary.json") as f:
    summary = json.load(f)
threshold = summary["best"]["threshold"]

# Fit scaler and PCA on training normals — must match what was used during training
extractor = EmbeddingExtractor(model, device=device, pooling="cls")
train_loader = DataLoader(train_normal, batch_size=512, shuffle=False, num_workers=2)
train_emb, train_lbl, _ = extractor.extract_from_dataloader(train_loader, return_labels=True, verbose=False)
train_emb = train_emb[train_lbl == 0]
sc = StandardScaler()
pca = PCA(n_components=130, random_state=42)
pca.fit_transform(sc.fit_transform(train_emb))

# Evaluate across all three test splits
for split in ["test_balanced", "test_strong", "test_subtle"]:
    ds = DroneChunkDataset("processed_data", split=split, return_labels=True)
    dl = DataLoader(ds, batch_size=512, shuffle=False, num_workers=2)
    emb_s, lbl_s, _ = extractor.extract_from_dataloader(dl, return_labels=True, verbose=False)
    emb_st = pca.transform(sc.transform(emb_s))
    scores_s = -lof.score_samples(emb_st)
    preds_s = (scores_s >= threshold).astype(int)
    tn_s, fp_s, fn_s, tp_s = confusion_matrix(lbl_s, preds_s).ravel()
    auc_s = roc_auc_score(lbl_s, scores_s)
    print(f"\n{split}:")
    print(classification_report(lbl_s, preds_s, target_names=["Normal", "Anomaly"]))
    print(f"AUC={auc_s:.4f}  Recall={tp_s/(tp_s+fn_s):.4f}  FAR={fp_s/(fp_s+tn_s):.4f}")
    print(f"Anomalies caught: {tp_s} / {tp_s+fn_s}")
```

## 🧪 Experiment Journey

### Phase 1 — Baseline: MLP Encoder + Isolation Forest

All early experiments used a 2-layer MLP JEPA encoder feeding into Isolation Forest.

| Configuration | Features | Detector | AUC | Recall | F1 | FAR |
|---|---|---|---|---|---|---|
| MLP encoder (standard) | 11 | IF | 0.4848 | 0.0248 | 0.0480 | 0.0482 |
| MLP encoder (alt. run) | 11 | IF | 0.4117 | 0.0191 | 0.0370 | 0.0494 |
| Deeper MLP + block masking | 11 | IF | 0.4592 | 0.0464 | 0.0877 | 0.0488 |
| Enhanced feature set | 17 | IF | 0.4562 | 0.0288 | 0.0555 | 0.0500 |
| Reconstruction-error scoring | 17 | Recon | 0.4363 | 0.0253 | 0.0489 | 0.0491 |
| Transformer encoder | 17 | IF | 0.3912 | 0.0226 | 0.0437 | 0.0520 |

All MLP + IF combinations were stable but severely under-sensitive (Recall < 5%). The bottleneck was representation quality, not the detector — motivating the switch to a Transformer encoder and LOF.

### Phase 2 — Optimization: JEPA v3 Transformer + LOF

After switching to the JEPA v3 Transformer encoder (4 attention heads, 2 layers, 384-dim output) and replacing IF with Local Outlier Factor, performance jumped significantly.

| Configuration | AUC | Recall | F1 | FAR | Score |
|---|---|---|---|---|---|
| IF + JEPA setup | 0.6587 | 0.3820 | 0.5348 | 0.1822 | 1.9101 |
| IF tuned | 0.6657 | 0.3935 | 0.5469 | 0.1673 | 2.0395 |
| LOF (Euclidean) | 0.6875 | 0.4130 | 0.5765 | 0.1134 | 2.2657 |
| LOF + StandardScaler + PCA128 + k=20 | 0.7687 | 0.4039 | 0.5700 | 0.0457 | 2.2730 |
| **LOF + Manhattan + PCA130 + k=21 ✓** | **0.7785** | **0.5314** | **0.6845** | **0.0833** | **2.60** |

**Why each change helped:**
- **LOF over IF:** Isolation Forest uses global path lengths, making it less sensitive to local density variation. LOF explicitly compares each point's density to its k-neighborhood — better suited for structured telemetry embeddings.
- **Manhattan over Euclidean:** Tampering typically affects a small number of feature dimensions sharply. Manhattan sums absolute differences equally; Euclidean squares them, diluting small but meaningful deviations.
- **PCA compression:** Reduces 384 JEPA dimensions to ~130, removing noisy trailing components that confuse LOF density estimates.
- **225-config sweep (Cell 62):** Tested 5 PCA dims × 3 distance metrics × 5 k-values × 2 scalers, all calibrated under FAR ≤ 20% on the validation set.

### Phase 3 — Per-Anomaly-Type Analysis

| Anomaly Type | Detection | Reason |
|---|---|---|
| altitude_spike, injection, coordinate_jump, combined | Easy (AUC > 0.83) | Large multi-dimensional deviations, clearly isolated in embedding space |
| speed_inconsistency, heading_inconsistency, deletion_gap | Medium | Moderate deviations |
| precision_rounding, timestamp_drift | Hard (AUC ≈ 0.50) | Nearly indistinguishable from normal in JEPA embedding space |

`timestamp_drift` is the hardest case — Recall ≈ 7%. It produces minimal embedding deviation, suggesting future work needs temporal features or longer context windows.

## 💡 Limitations & Future Work

- Chunk size of 20 waypoints is too short to capture longer-range flight semantics — exploring 50–100 waypoint windows with dilated attention
- Masking is currently random — semantic masking (masking related feature groups together) could yield richer representations
- `timestamp_drift` anomalies remain nearly undetectable (AUC ≈ 0.50) — temporal features or longer context are needed
- Adaptive online thresholding could improve robustness across different drone platforms and environments
